# 尾插法建表第三层图谱说明

最后更新：2026-03-19

## 1. 目标

本文件用于说明“`linked-list-tail-insert` / 尾插法建表”这一第三层节点的来源、拆分依据与依赖关系设计。

本次目标是把“链表”二级图谱中的“尾插法建表”继续拆到“尾指针维护、尾部链接和顺序保持”这一层，让尾插法与头插法形成更直观的对照。

## 2. 材料来源

- `docs/Table_of_Contents_for_Data_Structures.pdf`

直接依据：

- 第二章 2.3.2 单链表的基本操作实现
- 尾插法建立链表
- 单链表的相关概念

## 3. 第三层节点

当前拆成以下 8 个节点：

1. 头结点与尾指针
2. 新结点申请
3. 新结点数据赋值
4. 新结点 next 置空
5. 尾结点指向新结点
6. 尾指针后移
7. 保持输入顺序
8. 应用入口

## 4. 设计价值

这一层适合答辩时说明：

- 为什么尾插法的关键不是“找到末尾”，而是持续维护尾指针
- 为什么尾插法和头插法在结果顺序上正好形成对照
- 为什么这一层能把链表建表过程讲得更接近代码实现

## 5. 当前边界

本次只新增展示层第三层图谱，不改变路径规划算法和数据库结构。
