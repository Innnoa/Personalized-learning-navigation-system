## 4. 依赖设计原则

本细化图谱按“先理解约束结构，再理解线性化排序，最后进入工期分析”的学习顺序组织。

当前依赖主线如下：

- 有向无环图 -> AOV 网 / AOE 网
- AOV 网 -> 拓扑排序定义 -> 拓扑排序算法
- AOE 网 -> 关键路径概念 -> 关键路径求解
- 拓扑排序算法 -> 关键路径求解 / 应用入口
- 关键路径概念 / 关键路径求解 -> 应用入口

这样处理的原因是：

- 拓扑排序的成立必须建立在 DAG 这一结构前提之上
- AOV 网强调活动间的先后依赖，是讲清拓扑排序最合适的建模方式
- AOE 网强调事件和工期，更适合作为关键路径分析的理论基础
- 关键路径求解通常要依赖拓扑序，因此应明确把两者连接起来

## 6. 对论文与答辩的价值

拓扑排序与关键路径这一层细化适合用来展示系统如何把“图的基础表示与遍历”进一步推进到“任务依赖分析与工程调度”。

在演示时，可以从课程主图中的“拓扑排序与关键路径”节点双击进入，讲清楚：

- 为什么只有 DAG 才能进行稳定的拓扑排序
- 为什么 AOV 网和 AOE 网虽然都属于有向图，但建模对象不同
- 为什么拓扑排序算法是关键路径求解的重要前置步骤
- 为什么关键路径能直接服务于课程安排、工程进度和任务调度这类现实问题
