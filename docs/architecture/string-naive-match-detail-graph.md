# 简单模式匹配第三层图谱说明

最后更新：2026-03-19

## 1. 目标

本文件用于说明“`string-naive-match` / 简单模式匹配”这一第三层节点的来源、拆分依据与依赖关系设计。

本次目标是把“串”二级图谱中的“简单模式匹配”继续拆到“对齐、逐字符比较、失配右移与重复比较代价”这一层，为后面的 KMP 主过程建立明确对照。

## 2. 材料来源

- `docs/Table_of_Contents_for_Data_Structures.pdf`

直接依据：

- 第四章 4.3 串的模式匹配
- 简单模式匹配
- 失配后重新对齐

## 3. 第三层节点

当前拆成以下 8 个节点：

1. 主串与模式串
2. 起始位置对齐
3. 逐字符比较
4. 匹配成功条件
5. 失配判定
6. 模式串右移一位
7. 重复比较代价
8. 应用入口

## 4. 设计价值

这一层适合答辩时说明：

- 为什么朴素匹配是理解 KMP 的必要前置，而不是可以直接跳过的“旧算法”
- 为什么“模式串右移一位”看似简单，却会带来大量重复比较
- 为什么补上这一层后，串章节和 KMP 章节之间的过渡会更自然

## 5. 当前边界

本次只新增展示层第三层图谱，不改变路径规划算法和数据库结构。
