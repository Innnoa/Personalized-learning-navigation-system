## 4. 依赖设计原则

本细化图谱按“先理解图是什么，再理解如何表示，最后进入遍历算法与应用”的学习顺序组织。

当前依赖主线如下：

- 图定义与基本术语 -> 图的分类与连通性 / 图的抽象数据类型
- 图的分类与连通性 / 图的抽象数据类型 -> 邻接矩阵存储 / 邻接表存储
- 邻接矩阵存储 / 邻接表存储 -> 深度优先遍历 / 广度优先遍历
- 深度优先遍历 / 广度优先遍历 -> 应用入口

这样处理的原因是：

- 图的分类和连通性会直接影响对图结构特征和表示方式的理解
- 邻接矩阵与邻接表是后续遍历算法的两种最核心实现背景
- DFS 和 BFS 是后续连通性分析、拓扑排序和关键路径问题的共同基础

## 5. 当前实现约定

- 顶层课程图谱仍然来自 SQLite 中的 `knowledge_points` 与 `knowledge_dependencies`
- 细化图谱暂通过 `backend/config/knowledge_graph_details.json` 维护
- 前端在课程主图中双击“图的存储与遍历”节点，可进入该节点的细化图谱
- 页面上方面包屑会显示当前已进入 `graph-basic` scope，并支持返回课程主图
- 当前该细化节点只服务于图谱展示，不直接改变路径规划的课程级主线

## 6. 对论文与答辩的价值

图的存储与遍历这一层细化适合用来展示系统如何从线性结构、树结构进一步过渡到更复杂的网络结构。

在演示时，可以从课程主图中的“图的存储与遍历”节点双击进入，讲清楚：

- 为什么图比树更适合表达复杂关系网络
- 为什么邻接矩阵和邻接表会带来不同的空间与遍历代价
- 为什么 DFS 更贴近“沿路深入”的搜索思路，而 BFS 更贴近“分层扩展”的搜索思路
- 为什么这一层是后续“拓扑排序与关键路径”的共同前置基础

## 7. 当前边界

本次只完成了第十三个节点“图的存储与遍历”的细化图谱，不调整当前路径规划算法、学习记录逻辑和推荐解释生成逻辑。

也就是说：

- 当前“多层图谱下钻”仍先服务于知识图谱展示
- 路径规划仍以课程级主节点为基本单位
- 后续若继续细化，可以把 `topological-sort` 再单独展开为图算法方向的更深层图谱
