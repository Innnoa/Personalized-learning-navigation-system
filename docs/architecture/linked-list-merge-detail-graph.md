# 链表有序表合并第三层图谱说明

最后更新：2026-03-19

## 1. 目标

本文件用于说明“`linked-list-merge` / 有序表合并”这一第三层节点的来源、拆分依据与依赖关系设计。

本次目标是把“链表”二级图谱中的“有序表合并”继续拆到“当前结点比较、结果链重连、剩余链段接入与不搬移元素优势”这一层，让链表合并与顺序表合并形成清晰对照。

## 2. 材料来源

- `docs/Table_of_Contents_for_Data_Structures.pdf`

直接依据：

- 第二章 2.3.2 单链表的基本操作实现
- 基于单链表的有序表合并

## 3. 第三层节点

当前拆成以下 8 个节点：

1. 有序链表前提
2. 结果链表头结点
3. 比较当前结点
4. 较小结点接到结果链
5. 移动工作指针
6. 接上剩余链段
7. 不搬移元素优势
8. 应用入口

## 4. 设计价值

这一层适合答辩时说明：

- 为什么链表合并最核心的不是复制元素，而是重连指针
- 为什么它和顺序表合并在“实现手段”上形成强烈对照
- 为什么这一层特别适合展示链式结构在局部修改上的优势

## 5. 当前边界

本次只新增展示层第三层图谱，不改变路径规划算法和数据库结构。
