# 复杂度案例解析第三层图谱说明

## 1. 目标

本文件用于说明“`algorithm-analysis-case-study` / 复杂度案例解析”这一第三层节点的来源、拆分依据与依赖关系设计。

本次目标是把“算法复杂度分析”二级图谱中的“复杂度案例解析”继续拆到“识别基本操作、建立频度表达式、比较不同案例结果”这一层，让复杂度分析过程具备更直接的演示价值。

## 3. 第三层节点

当前拆成以下 8 个节点：

1. 识别基本操作
2. 建立频度表达式
3. 单层循环案例
4. 双层循环案例
5. 条件分支影响

## 4. 设计价值

这一层适合答辩时说明：

- 为什么复杂度分析不是直接写结论，而是要先找基本操作
- 为什么单层循环、双层循环和条件分支会带来不同复杂度结果
- 为什么这一层能把“时间复杂度”从抽象概念落到可操作的分析步骤
