# 关键路径求解第三层图谱说明

## 1. 目标

本文件用于说明“`topological-sort-critical-path-solve` / 关键路径求解”这一第三层节点的来源、拆分依据与依赖关系设计。

本次目标是把“拓扑排序与关键路径”二级图谱中的“关键路径求解”继续拆到“ve/vl、e/l、时间余量、关键活动和关键路径形成”这一层，使第六章最贴近工程调度的算法过程能够被逐步解释。

## 3. 第三层节点

当前拆成以下 8 个节点：

1. 正向计算 ve
2. 逆向计算 vl
3. 活动最早开始 e
4. 活动最迟开始 l
5. 时间余量
6. 关键活动判定
7. 关键路径形成
8. 调度应用入口

这一层适合答辩时说明：

- 为什么关键路径求解并不是一句“找最长路径”这么简单

- 为什么这一层与项目“个性化学习导航”的依赖排序思想天然呼应
