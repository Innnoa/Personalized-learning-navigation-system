## 4. 依赖设计原则

本细化图谱按“先理解结点和概念，再掌握操作，再进入建表、应用与比较”的学习顺序组织。

当前依赖主线如下：

- 结点结构 -> 相关概念 / 查找操作
- 相关概念 / 查找操作 -> 插入操作 / 删除操作
- 插入操作 -> 头插法建表 / 尾插法建表
- 删除操作 / 尾插法建表 -> 有序表合并
- 查找操作 / 插入操作 -> 与顺序表比较 -> 其他链表形式

这样处理的原因是：

- 链表所有操作都建立在“结点 + 指针关系”的理解之上
- 头插法、尾插法本质是插入操作在建表场景下的具体化
- “与顺序表比较”能把链表的优势和限制说清楚，也适合作为后续扩展到双向链表、循环链表的过渡节点

## 5. 当前实现约定

- 页面上方面包屑会显示当前已进入 `linked-list` scope，并支持返回课程主图
- 当前该细化节点只服务于图谱展示，不直接改变路径规划的课程级主线

## 6. 对论文与答辩的价值

链表这一层细化适合用来展示系统如何处理“与顺序表形成对照”的知识结构。

在演示时，可以从“线性表”主线继续进入“链表”，讲清楚：

- 为什么链表不支持随机访问
- 为什么插入删除时链表通常比顺序表更灵活
- 为什么头插法和尾插法能体现不同建表策略
- 为什么循环链表、双向链表是对基本链表的自然扩展

这能明显增强知识图谱展示的教学解释力。
