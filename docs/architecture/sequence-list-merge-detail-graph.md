# 顺序表合并第三层图谱说明

最后更新：2026-03-19

## 1. 目标

本文件用于说明“`sequence-list-merge` / 合并操作”这一第三层节点的来源、拆分依据与依赖关系设计。

本次目标是把“顺序表”二级图谱中的“合并操作”继续拆到“有序前提、双指针扫描、结果有序保持与代价分析”这一层，让顺序表合并过程能被单独展示。

## 3. 第三层节点

当前拆成以下 8 个节点：

1. 有序前提
2. 结果表空间准备
3. 双指针比较
4. 较小元素先写入
5. 复制剩余元素
6. 结果有序保持
7. 时间与空间代价
8. 应用入口

## 4. 设计价值

这一层适合答辩时说明：

- 为什么顺序表合并通常默认建立在“两个有序表”前提下
- 为什么双指针扫描是教材里最经典的线性合并思路
- 为什么这一层能自然过渡到归并排序等更复杂算法

## 5. 当前边界

本次只新增展示层第三层图谱，不改变路径规划算法和数据库结构。
