# 数据结构基本概念细化图谱说明

最后更新：2026-03-19

## 1. 目标

本文件用于说明“`ds-intro` / 数据结构基本概念”这一课程级节点的第一版细化图谱来源、拆分依据与依赖关系设计。

当前目标不是一次性把整门课全部拆到最细，而是先把第一个节点的“多层图谱下钻”链路跑通，作为后续其他章节继续细化的模板。

## 2. 材料来源

当前细化图谱主要依据以下本地材料整理：

- `docs/Table_of_Contents_for_Data_Structures.pdf`

其中与本节点直接对应的目录依据为：

- 第一章 绪论
- 1.2 基本概念和术语
- 1.2.1 数据、数据元素、数据项和数据对象
- 1.2.2 数据结构、逻辑结构和存储结构
- 1.2.3 数据类型和抽象数据类型

## 3. 第一版细化节点

当前把“数据结构基本概念”拆成以下 9 个细化节点：

1. 数据
2. 数据元素
3. 数据项
4. 数据对象
5. 数据结构
6. 逻辑结构
7. 存储结构
8. 数据类型
9. 抽象数据类型

## 4. 依赖设计原则

本细化图谱不是照搬教材目录顺序，而是按“学习时先理解哪个概念，再进入哪个概念”的依赖关系组织。

当前依赖主线如下：

- 数据 -> 数据元素 -> 数据项
- 数据元素 -> 数据对象
- 数据元素 / 数据对象 -> 数据结构
- 数据结构 -> 逻辑结构
- 数据结构 -> 存储结构
- 数据 -> 数据类型
- 数据类型 / 数据结构 -> 抽象数据类型

## 5. 当前实现约定

- 顶层课程图谱仍然来自 SQLite 中的 `knowledge_points` 与 `knowledge_dependencies`
- 细化图谱暂通过 `backend/config/knowledge_graph_details.json` 维护
- 前端在课程主图中双击带“可细化”标记的节点，可进入该节点的细化图谱
- 页面上方使用面包屑表示当前图谱路径，支持随时返回上层
- 当前已实现的第一个细化 scope 为：`ds-intro`
- 当前已支持继续双击“逻辑结构”“存储结构”进入第三层图谱

## 6. 后续扩展方式

后续每个章节节点继续细化时，建议沿用同一模板：

1. 先从课程目录中确定该节点对应的小节范围
2. 再从题库或讲义中补充更细粒度知识点
3. 形成“节点清单 + 依赖关系 + 简要说明”
4. 追加到 `backend/config/knowledge_graph_details.json`
5. 若某个细化节点还需继续拆分，则为该节点再增加下一层 `detailScopeCode`

## 7. 当前边界

本次只完成了第一个节点“数据结构基本概念”的细化图谱，不改变现有路径规划算法和学习者画像建模范围。

也就是说：

- 当前“多层图谱下钻”先服务于知识图谱展示
- 路径规划仍以课程级主节点为基本单位
- 待后续更多章节细化完成后，再决定是否将更细粒度节点纳入规划算法
