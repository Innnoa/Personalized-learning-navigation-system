# 线性表顺序表示第三层图谱说明

最后更新：2026-03-19

## 1. 目标

本文件用于说明“`linear-list-sequential-representation` / 顺序表示”这一第三层节点的来源、拆分依据与依赖关系设计。

本次目标是把“线性表”二级图谱中的“顺序表示”继续拆到“连续空间、随机访问、容量约束和插删搬移”这一层，让顺序表示为什么会导向顺序表实现更容易展示。

## 3. 第三层节点

当前拆成以下 8 个节点：

1. 连续存储空间
2. 地址连续特征
3. 随机访问能力
4. 容量预分配
5. 插入删除搬移
6. 存储密度较高
7. 适用场景判断
8. 应用入口

## 4. 设计价值

这一层适合答辩时说明：

- 为什么顺序表示最重要的优势是随机访问
- 为什么顺序表示的主要短板是插入删除时的元素搬移
- 为什么这一层能自然过渡到后续“顺序表”二级图谱
