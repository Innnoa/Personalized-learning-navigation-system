# 关键路径概念第三层图谱说明

最后更新：2026-03-19

## 1. 目标

本文件用于说明“`topological-sort-critical-path-concept` / 关键路径概念”这一第三层节点的来源、拆分依据与依赖关系设计。

本次目标是把“拓扑排序与关键路径”二级图谱中的“关键路径概念”继续拆到“总工期、最长路径、关键活动、时间余量和求解关系”这一层，让“关键路径到底是什么”在进入计算步骤前就先讲明白。

## 3. 第三层节点

当前拆成以下 8 个节点：

1. 工程总工期
2. 最长路径含义
3. 关键活动定义
4. 时间余量为零
5. 路径决定工期
6. 非关键活动可延迟
7. 与关键路径求解关系
8. 应用入口

## 4. 设计价值

这一层适合答辩时说明：

- 为什么关键路径本质上是“决定工期的最长路径”，而不是普通路径概念
- 为什么“关键路径概念”和“关键路径求解”必须拆开
- 为什么这一层和本项目里的“关键前置依赖”解释逻辑很容易形成呼应

本次只新增展示层第三层图谱，不改变路径规划算法和数据库结构。
