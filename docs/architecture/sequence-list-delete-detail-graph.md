# 顺序表删除第三层图谱说明

最后更新：2026-03-19

## 1. 目标

本文件用于说明“`sequence-list-delete` / 删除操作”这一第三层节点的来源、拆分依据与依赖关系设计。

本次目标是把“顺序表”二级图谱中的“删除操作”继续拆到“空表检查、位置检查、前移覆盖和长度更新”这一层，让顺序表删除的核心代价和执行流程能够被单独展示。

## 3. 第三层节点

当前拆成以下 8 个节点：

1. 空表判定
2. 删除位置合法性
3. 记录被删元素
4. 后继元素逐位前移
5. 覆盖删除空位
6. 长度更新
7. 时间代价理解
8. 应用入口

## 4. 设计价值

这一层适合答辩时说明：

- 为什么顺序表删除和插入一样，真正的成本在元素搬移
- 为什么删除前要先记录被删元素，否则返回值会丢失
- 为什么这一层能更完整地支撑“顺序表适合查询多、修改少”的课程结论

本次只新增展示层第三层图谱，不改变路径规划算法和数据库结构。
