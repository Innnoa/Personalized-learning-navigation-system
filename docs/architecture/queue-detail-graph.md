## 4. 依赖设计原则

本细化图谱按“先理解先进先出约束，再理解顺序实现与循环改进，最后过渡到链式实现和应用”的学习顺序组织。

当前依赖主线如下：

- 基本概念 -> 抽象数据类型
- 抽象数据类型 -> 顺序队列 / 链队列
- 顺序队列 -> 循环队列 / 队列初始化
- 队列初始化 -> 入队操作 / 出队操作
- 循环队列 -> 入队操作 / 出队操作
- 入队操作 / 出队操作 / 链队列 -> 典型应用

这样处理的原因是：

- “顺序队列”到“循环队列”体现了教材中最重要的一次实现改进
- “链队列”能作为队列的另一种动态实现方向，与顺序实现形成对照
- “典型应用”适合把银行排队等调度问题和数据结构特征直接关联起来

## 6. 对论文与答辩的价值

队列这一层细化适合用来展示系统如何把“基础结构”直接映射到“现实调度问题”。

在演示时，可以从课程主图中的“队列”节点双击进入，讲清楚：

- 为什么队列满足先进先出
- 为什么普通顺序队列会出现空间复用问题
- 为什么循环队列能解决假溢出
- 为什么链队列适合规模动态变化的场景
- 为什么银行排队系统天然适合用队列建模
