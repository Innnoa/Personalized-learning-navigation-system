# 顺序表插入第三层图谱说明

最后更新：2026-03-19

## 1. 目标

本文件用于说明“`sequence-list-insert` / 插入操作”这一第三层节点的来源、拆分依据与依赖关系设计。

本次目标是把“顺序表”二级图谱中的“插入操作”继续拆到“位置检查、表满判定、元素后移、写入和长度更新”这一层，让顺序表插入为什么代价高、流程如何执行能够被单独展示。

## 3. 第三层节点

当前拆成以下 8 个节点：

1. 插入位置合法性
2. 表满判定
3. 末尾到前逐位后移
4. 空出插入位置
5. 写入新元素
6. 长度更新
7. 时间代价理解
8. 应用入口

## 4. 设计价值

这一层适合答辩时说明：

- 为什么顺序表插入的关键不是“赋值”，而是“元素搬移”
- 为什么必须从表尾向前移动，不能反过来做
- 为什么这一层能自然引出顺序表适合查询多、修改少的课程结论

本次只新增展示层第三层图谱，不改变路径规划算法和数据库结构。
