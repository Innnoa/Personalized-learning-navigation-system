# 二叉树遍历细化图谱说明

最后更新：2026-03-19

## 1. 目标

本文件用于说明“`binary-tree-traversal` / 二叉树遍历”这一课程级节点的第一版细化图谱来源、拆分依据与依赖关系设计。

本次目标是把“二叉树遍历”从课程主图中的算法型节点继续拆到“遍历定义、四种典型遍历、递归与非递归实现思想、应用入口”这一层，让系统在第五章中不仅能展示树结构基础，也能展示代表性树算法的学习路径。

## 3. 第一版细化节点

当前把“二叉树遍历”拆成以下 8 个细化节点：

1. 遍历定义与策略
2. 递归遍历思想
3. 先序遍历
4. 中序遍历
5. 后序遍历
6. 层序遍历
7. 非递归遍历思想
8. 应用入口

## 4. 依赖设计原则

本细化图谱按“先理解什么是遍历，再区分深度优先主线与层序主线，最后进入非递归实现和应用”的学习顺序组织。

当前依赖主线如下：

- 遍历定义与策略 -> 递归遍历思想 / 层序遍历
- 递归遍历思想 -> 先序遍历 / 中序遍历 / 后序遍历
- 先序遍历 / 中序遍历 / 后序遍历 -> 非递归遍历思想
- 层序遍历 / 非递归遍历思想 -> 应用入口

这样处理的原因是：

- “递归遍历思想”是先序、中序、后序三类深度优先遍历的共同前提
- “层序遍历”在概念上同样属于遍历，但实现主线更接近队列驱动的广度优先访问
- “非递归遍历思想”更适合放在掌握三种基本访问次序之后，用来说明如何把递归过程改写为显式数据结构控制

## 5. 当前实现约定

- 顶层课程图谱仍然来自 SQLite 中的 `knowledge_points` 与 `knowledge_dependencies`
- 细化图谱暂通过 `backend/config/knowledge_graph_details.json` 维护
- 前端在课程主图中双击“二叉树遍历”节点，可进入该节点的细化图谱
- 页面上方面包屑会显示当前已进入 `binary-tree-traversal` scope，并支持返回课程主图
- 当前该细化节点只服务于图谱展示，不直接改变路径规划的课程级主线

## 6. 对论文与答辩的价值

二叉树遍历这一层细化适合用来展示系统如何从“结构基础”过渡到“典型算法过程”，也方便在答辩时说明知识点依赖图不仅能画静态结构，还能支持算法学习顺序的分层表达。

在演示时，可以从课程主图中的“二叉树遍历”节点双击进入，讲清楚：

- 为什么遍历的核心是“访问次序”
- 为什么先序、中序、后序都依赖同一套递归框架
- 为什么层序遍历与前三者在思路和数据结构支持上不同
- 为什么非递归遍历能体现“栈和队列”与“树遍历算法”的跨章节关联

## 7. 当前边界

本次只完成了第十一个节点“二叉树遍历”的细化图谱，不调整当前路径规划算法、学习记录逻辑和推荐解释生成逻辑。

也就是说：

- 当前“多层图谱下钻”仍先服务于知识图谱展示
- 路径规划仍以课程级主节点为基本单位
- 后续若继续细化，可以把 `huffman-tree`、`threaded-binary-tree` 等节点继续展开为更深层算法图谱
