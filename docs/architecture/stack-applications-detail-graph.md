# 栈的典型应用第三层图谱说明

最后更新：2026-03-19

## 1. 目标

本文件用于说明“`stack-applications` / 典型应用”这一第三层节点的来源、拆分依据与依赖关系设计。

本次目标是把“栈”二级图谱中的“典型应用”继续拆到“数制转换、括号匹配、表达式处理、递归回溯与状态恢复”这一层，让栈为什么能用于这些问题具备可展示、可讲解的中间层。

这一层适合答辩时说明：

- 为什么栈的应用不是零散案例，而是都可归结为后进先出约束
- 为什么“表达式转换”和“表达式求值”适合拆成两个节点分别说明
- 为什么这一层能把栈从数据结构定义自然延伸到程序执行和真实场景
