# 递归工作栈第三层图谱说明

最后更新：2026-03-19

## 1. 目标

本文件用于说明“`stack-recursion` / 递归工作栈”这一第三层节点的来源、拆分依据与依赖关系设计。

本次目标是把“栈”二级图谱中的“递归工作栈”继续拆到“调用现场、工作栈、入栈建帧、出栈返回、递归与非递归转换”这一层，突出栈与递归之间的跨章节桥梁作用。

当前拆成以下 8 个节点：

1. 函数调用回顾
2. 调用现场
3. 系统工作栈
4. 入栈建帧
5. 终止条件

## 4. 设计价值

这一层适合答辩时说明：

- 为什么递归本质上仍然依赖栈
- 为什么树遍历、图搜索能回到“工作栈”这一共同机制
- 为什么第三层图谱不是随意细化，而是围绕真正值得解释的“关键机制节点”继续展开
