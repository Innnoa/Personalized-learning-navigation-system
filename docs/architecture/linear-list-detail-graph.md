# 线性表细化图谱说明

最后更新：2026-03-19

## 1. 目标

本文件用于说明“`linear-list` / 线性表”这一课程级节点的第一版细化图谱来源、拆分依据与依赖关系设计。

本次目标不是把顺序表、链表的所有实现细节都塞进同一层，而是先把“线性表”这一课程级节点拆到“概念、抽象接口、表示方向、典型问题”这一层，作为后续继续下钻的中间层。

## 3. 第一版细化节点

当前把“线性表”拆成以下 9 个细化节点：

1. 线性表定义
2. 线性表特点
3. 基本操作
4. 抽象数据类型
5. 合并问题
6. 合并算法设计
7. 顺序表示
8. 链式表示
9. 应用实例

## 4. 依赖设计原则

本细化图谱按“先理解对象与特点，再理解抽象接口，最后过渡到表示方式与应用问题”的学习顺序组织。

当前依赖主线如下：

- 线性表定义 -> 线性表特点 / 基本操作
- 线性表特点 / 基本操作 -> 抽象数据类型
- 抽象数据类型 -> 合并问题 -> 合并算法设计
- 抽象数据类型 -> 顺序表示 / 链式表示
- 顺序表示 / 链式表示 / 合并算法设计 -> 应用实例

这样处理的原因是：

- “抽象数据类型”是从概念过渡到实现与应用的桥梁
- “顺序表示”“链式表示”在这里是实现方向概览，不替代课程主图中的“顺序表”“链表”独立节点
- “合并问题”“合并算法设计”能体现教材中“基于 ADT 设计算法”的一条清晰示例链

## 5. 当前实现约定

- 顶层课程图谱仍然来自 SQLite 中的 `knowledge_points` 与 `knowledge_dependencies`
- 细化图谱暂通过 `backend/config/knowledge_graph_details.json` 维护
- 前端在课程主图中双击“线性表”节点，可进入该节点的细化图谱
- 页面上方面包屑会显示当前已进入 `linear-list` scope，并支持返回课程主图
- 当前该细化节点只服务于图谱展示，不直接改变路径规划的课程级主线
- 当前已支持继续双击“顺序表示”“链式表示”进入第三层图谱

## 6. 对论文与答辩的价值

这组细化节点适合答辩展示的原因在于，它能把“课程主图中的一个章节节点”继续拆成更接近教学逻辑的概念链，而不是简单复制目录标题。

在演示时，可以从课程主图中的“线性表”节点双击进入，说明系统如何把：

- 线性表定义
- 抽象数据类型
- 顺序/链式表示
- 合并算法与应用实例

组织成一条可解释的知识依赖链。

## 7. 当前边界

本次只完成了第三个节点“线性表”的细化图谱，不调整当前路径规划算法、学习记录逻辑和推荐解释生成逻辑。

也就是说：

- 当前“多层图谱下钻”仍先服务于知识图谱展示
- 路径规划仍以课程级主节点为基本单位
- 后续若继续做更细层级，可把 `linear-list-sequential-representation` 或 `linear-list-linked-representation` 继续挂接下一层 scope
