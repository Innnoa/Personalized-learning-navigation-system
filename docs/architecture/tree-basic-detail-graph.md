本文件用于说明“`tree-basic` / 树与二叉树基础”这一课程级节点的第一版细化图谱来源、拆分依据与依赖关系设计。

## 4. 依赖设计原则

本细化图谱按“先理解一般树，再理解二叉树，再理解其性质、存储和后续用途”的学习顺序组织。

当前依赖主线如下：

- 树的定义与术语 -> 树的抽象数据类型 / 二叉树定义与特点
- 二叉树定义与特点 -> 二叉树抽象数据类型 / 二叉树性质 / 树与二叉树关系
- 二叉树抽象数据类型 / 二叉树性质 -> 二叉树存储结构
- 树的抽象数据类型 / 二叉树定义与特点 -> 树与二叉树关系
- 二叉树存储结构 / 树与二叉树关系 -> 应用入口

这样处理的原因是：

- “树”和“二叉树”既有联系也有区别，不能直接跳过一般树概念
- “二叉树性质”是后续遍历和编码类问题的重要数学基础
- “树与二叉树关系”能为后续森林转换、哈夫曼树等内容提供解释桥梁

## 5. 当前实现约定

- 顶层课程图谱仍然来自 SQLite 中的 `knowledge_points` 与 `knowledge_dependencies`
- 细化图谱暂通过 `backend/config/knowledge_graph_details.json` 维护
- 前端在课程主图中双击“树与二叉树基础”节点，可进入该节点的细化图谱
- 页面上方面包屑会显示当前已进入 `tree-basic` scope，并支持返回课程主图
- 当前该细化节点只服务于图谱展示，不直接改变路径规划的课程级主线

## 6. 对论文与答辩的价值

树与二叉树基础这一层细化适合用来展示系统如何处理“从一般结构到特殊结构”的知识过渡。

在演示时，可以从课程主图中的“树与二叉树基础”节点双击进入，讲清楚：

- 为什么树用于表达层次关系
- 为什么二叉树是更适合算法处理的一类重要树结构
- 为什么二叉树性质和存储方式会影响后续遍历和编码算法
- 为什么这一层是“二叉树遍历”和“哈夫曼树与编码”的共同前置基础

本次只完成了第十个节点“树与二叉树基础”的细化图谱，不调整当前路径规划算法、学习记录逻辑和推荐解释生成逻辑。
