# KMP 模式匹配细化图谱说明

最后更新：2026-03-19

## 1. 目标

本文件用于说明“`kmp` / KMP 模式匹配”这一课程级节点的第一版细化图谱来源、拆分依据与依赖关系设计。

本次目标是把“KMP 模式匹配”从课程主图中的算法节点继续拆到“问题动机、部分匹配思想、next 数组、失配回退、主过程、复杂度优势”这一层，形成第四章里最具有算法解释价值的一条下钻链。

## 3. 第一版细化节点

当前把“KMP 模式匹配”拆成以下 8 个细化节点：

1. 模式匹配回顾
2. 朴素匹配局限
3. 部分匹配思想
4. next 数组含义
5. next 求法
6. 失配回退机制
7. KMP 主过程
8. 复杂度优势

## 4. 依赖设计原则

本细化图谱按“先明确为什么需要 KMP，再理解它如何工作，最后解释它为什么更快”的学习顺序组织。

当前依赖主线如下：

- 模式匹配回顾 -> 朴素匹配局限 -> 部分匹配思想
- 部分匹配思想 -> next 数组含义 -> next 求法 / 失配回退机制
- next 求法 / 失配回退机制 -> KMP 主过程 -> 复杂度优势
- 朴素匹配局限 -> 复杂度优势

这样处理的原因是：

- KMP 的难点不在于代码步骤本身，而在于“为什么只回退模式串”这一思想转变
- `next` 数组是连接“思想”和“算法执行”的核心桥梁
- “复杂度优势”必须建立在和朴素匹配对照的基础上才容易解释清楚

## 5. 当前实现约定

- 顶层课程图谱仍然来自 SQLite 中的 `knowledge_points` 与 `knowledge_dependencies`
- 细化图谱暂通过 `backend/config/knowledge_graph_details.json` 维护
- 前端在课程主图中双击“KMP 模式匹配”节点，可进入该节点的细化图谱
- 页面上方面包屑会显示当前已进入 `kmp` scope，并支持返回课程主图
- 当前该细化节点只服务于图谱展示，不直接改变路径规划的课程级主线

## 6. 对论文与答辩的价值

KMP 这一层细化非常适合答辩展示，因为它能体现系统不仅会展示章节概念，还能展开到算法内部机制。

在演示时，可以从课程主图中的“KMP 模式匹配”节点双击进入，讲清楚：

- 为什么朴素匹配会重复比较
- 为什么部分匹配思想能减少回溯
- 为什么 `next` 数组是 KMP 的关键
- 为什么 KMP 主过程中主串指针不回退
- 为什么最终复杂度能降低到线性量级

## 7. 当前边界

本次只完成了第九个节点“KMP 模式匹配”的细化图谱，不调整当前路径规划算法、学习记录逻辑和推荐解释生成逻辑。

也就是说：

- 当前“多层图谱下钻”仍先服务于知识图谱展示
- 路径规划仍以课程级主节点为基本单位
- 后续若继续细化，可以把 `kmp-next-build` 或 `kmp-main-process` 继续拆到更细的 next 迭代过程与示例推演
