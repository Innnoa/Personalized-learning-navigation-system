# 时间复杂度第三层图谱说明

最后更新：2026-03-19

## 1. 目标

本文件用于说明“`algorithm-analysis-time-complexity` / 时间复杂度”这一第三层节点的来源、拆分依据与依赖关系设计。

本次目标是把“算法复杂度分析”二级图谱中的“时间复杂度”继续拆到“问题规模、渐近思想、大 O 记号、复杂度级别与增长比较”这一层，让复杂度分析不再只是一个名词，而是能被逐步讲清的知识链。

## 3. 第三层节点

当前拆成以下 8 个节点：

1. 时间复杂度目标
2. 问题规模 n
3. 忽略常数与低阶项
4. 大 O 记号
5. 最好最坏平均情况
6. 常见复杂度级别
7. 渐进增长比较
8. 应用入口

## 4. 设计价值

这一层适合答辩时说明：

- 为什么复杂度分析不是在背公式，而是在比较增长趋势
- 为什么“大 O”必须建立在问题规模和渐近思想之上
- 为什么这一层可以作为后续解释顺序表插入、图遍历和排序代价差异的理论入口
