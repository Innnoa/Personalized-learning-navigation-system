# 课程范围说明

最后更新：2026-03-17

## 1. 文档用途

本文件用于明确当前毕设演示所选课程的边界，避免系统能力设计过大、数据采集范围失控、论文叙述对象不清。

## 2. 当前约定

- 第一阶段只围绕一门课程做完整演示
- 课程名称当前待定，确认后应第一时间补充到本文件
- 后续数据库样例数据、知识点依赖图、推荐路径演示，都必须以本文件约定的课程范围为准

## 3. 课程选择建议

- 优先选择结构清晰、章节递进明显、知识点依赖关系容易说明的课程
- 优先选择自己熟悉、资料容易整理的课程
- 第一版演示建议控制在 15 到 30 个核心知识点，避免图谱过大影响实现与答辩展示

可选示例：

- C++ 程序设计基础
- 数据结构基础
- 计算机网络基础

## 4. 确认后需要补充的信息

- 课程名称
- 面向学习者对象
- 课程总目标
- 课程章节范围
- 核心知识点列表
- 典型前置依赖关系
- 适合演示的学习目标场景

## 5. 与系统设计的关系

- `database/seed/` 中的课程样例数据必须与本文件保持一致
- 知识点数量与依赖关系复杂度，应以“本科毕设可演示”为上限
- 如果后续切换课程或扩大范围，必须先更新本文件，再调整数据库与算法设计
