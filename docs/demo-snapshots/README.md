# Demo Snapshots

本目录用于存放答辩展示时的固定截图。

目标不是“临场随便截几张”，而是提前固定截图顺序、命名和展示重点，让 PPT、论文附录和现场演示三者保持一致。

## 1. 建议截图环境

- 推荐先执行：`DEMO_CHECK_AUTO_DOWN=0 ./scripts/demo_check.sh`
- 推荐使用独立演示实例：
  - 前端：`http://127.0.0.1:5174`
  - 后端：`http://127.0.0.1:18080`
- 推荐截图窗口宽度：`1440` 到 `1600` 像素
- 推荐统一使用默认学习者：`demo-learner`
- 推荐统一围绕“拓扑排序与关键路径”这一主链路截图，避免图谱范围频繁切换

## 2. 固定截图清单

建议按以下文件名保存，后续 PPT、论文附录和演示说明都直接引用这些名称。

### 01-home-ready.png

- 页面：首页
- 操作：刚启动演示环境后截图
- 必须包含：
  - 顶部导航
  - 健康状态正常
  - 首页主布局完整
- 用途：
  - 证明系统可启动、可进入演示状态

### 02-learning-graph-target-selection.png

- 页面：学习图谱
- 操作：
  1. 进入“学习图谱”
  2. 选中 `topological-sort / 拓扑排序与关键路径`
  3. 展示节点详情区
- 必须包含：
  - 当前节点高亮
  - 面包屑 / 当前范围
  - “设为当前学习目标”按钮
- 用途：
  - 说明系统支持由图谱反向驱动路径规划

### 03-home-path-generated.png

- 页面：首页
- 操作：
  1. 从学习图谱返回首页
  2. 点击“生成学习路径”
- 必须包含：
  - 目标节点
  - 路径总说明
  - 本轮推荐学习
  - 下一轮建议
- 用途：
  - 作为论文中“个性化学习路径生成结果”核心截图

### 04-home-feedback-compare.png

- 页面：首页
- 操作：
  1. 对本轮节点提交一条反馈
  2. 展示“路径前后对比区”
- 必须包含：
  - 反馈快捷录入
  - 调整前后路径变化
  - 最近一次反馈或回退状态
- 用途：
  - 说明系统支持动态调整与回退

### 05-resource-page-main.png

- 页面：推荐资源
- 操作：
  1. 从首页路径项点击“查看推荐资源”
  2. 保持一级资源页展开
- 必须包含：
  - 当前目标节点
  - 资源卡片
  - 资源来源 / 阶段 / 选择原因
- 用途：
  - 说明学习导航不只给路径，还给配套资源

### 06-detail-learning-view.png

- 页面：细化学习
- 操作：
  1. 进入 `topological-sort` 细化学习
  2. 选中 `topological-sort-algorithm`
  3. 生成细化路径
- 必须包含：
  - 细化图谱
  - 当前目标节点
  - 细化路径规划结果
- 用途：
  - 说明系统支持二级范围内继续导航，而不是只停留在课程级节点

### 07-resource-page-detail.png

- 页面：推荐资源
- 操作：
  1. 从细化学习页面点击“查看推荐资源”
  2. 保持二级资源页展开
- 必须包含：
  - 当前细化范围
  - 当前二级节点资源
  - 与首页资源页不同的上下文
- 用途：
  - 说明资源推荐与细化学习场景联动

### 08-learner-profile.png

- 页面：学习者画像
- 操作：
  1. 完成一轮反馈后进入画像页
  2. 展示掌握状态、画像统计和近期记录
- 必须包含：
  - 掌握度色阶
  - 画像统计模块
  - 近期学习记录
- 用途：
  - 说明系统具备结果沉淀与画像反馈能力

## 3. PPT 和论文建议用法

- PPT 正文建议使用 `03 / 04 / 05 / 06 / 08`
- 论文正文建议使用 `03 / 06 / 08`
- `01 / 02 / 07` 更适合放在附录或答辩备用页

## 4. 截图时的注意事项

- 截图前先执行一次演示重置，避免掌握度和记录状态漂移
- 尽量保持浏览器缩放为 `100%`
- 截图时不要让下拉菜单、悬浮提示或鼠标选中文本遮挡主区域
- 同一页如果需要重新截图，沿用同一文件名覆盖，避免 PPT 与论文引用漂移
