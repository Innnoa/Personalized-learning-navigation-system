《基于路径规划的个人学习导航系统》

一、实现平台与技术

1、前端：Vue3 + Vite + JavaScript + Cytoscape.js。  
2、后端：Drogon + C++17 + CMake。  
3、数据库：SQLite。  
4、核心实现：知识点依赖图建模、个性化学习路径规划、练习反馈动态调整、推荐理由解释、推荐资源联动与学习者画像展示。

本系统采用前后端分离的实现方式。前端负责知识图谱展示、学习目标选择、路径结果展示、练习反馈录入、推荐资源查看和学习者画像呈现；后端负责知识图谱读取、学习路径规划、反馈处理、推荐理由生成与画像聚合；SQLite 负责存储课程、知识点、依赖关系、学习者掌握度、反馈记录与资源行为记录。系统整体遵循“本地可部署、结构清晰、便于讲解、适合答辩演示”的设计原则，符合本科毕业设计对可实现性和可展示性的要求。

二、系统需求分析与设计

1、背景及意义

在数据结构等课程的学习过程中，知识点之间往往存在明显的前置依赖关系。学生在实际学习中经常会遇到以下问题：一是只知道当前某个知识点不会，但不知道应先补哪一部分前置基础；二是缺乏合理的学习顺序安排，容易跳过基础内容直接学习后续知识；三是学习过程中缺少根据掌握情况动态调整学习路径的机制；四是即使知道当前该学什么，也难以及时找到与当前阶段匹配的学习资源。因此，构建一个能够结合知识点依赖关系、学习者掌握度与时间预算，生成个性化学习路径的系统，具有较强的现实意义。

本项目面向个人学习导航场景，围绕课程知识图谱、路径规划、练习反馈、资源推荐和学习者画像五条主线展开。系统通过将课程知识点建模为依赖图，并结合学习者当前掌握度、目标节点和可用时间，生成分阶段的学习路径；在学习者提交练习反馈后，系统还可以动态调整路径结果，使推荐更加贴近真实学习进展。该系统既能够帮助学习者提高学习效率，也为后续论文撰写和答辩展示提供了较完整的实现基础。

2、应用现状

当前在线学习平台大多能够提供课程资源、章节目录和基本学习记录，但多数系统更偏向“资源汇总”，对“学习导航”的支持相对不足。对于依赖关系较强的课程，仅依靠章节顺序并不能满足个性化学习需求，学习者仍需自行判断应先学习哪个知识点、后学习哪个知识点。近年来，知识图谱、个性化推荐和自适应学习逐渐成为研究热点，但很多方案实现复杂度较高，或者更偏研究验证，缺少面向本科毕业设计场景的可落地系统。

相较于单纯的静态推荐，本项目更强调以下几个方面：一是显式表达知识点之间的依赖关系；二是在路径生成时同时考虑学习目标、掌握度和时间预算；三是推荐理由必须来源于路径规划过程中的真实依据，而不是凭空生成；四是支持练习反馈后的动态调整；五是在给出学习路径的同时提供与当前阶段相匹配的推荐资源和学习画像展示。上述设计使系统更符合个人学习导航系统的实际需求，也更适合作为本科毕业设计进行完整实现。

3、系统用例分析

本系统当前主要面向单学习者演示场景，核心角色包括学习者和系统维护者。

学习者的主要操作包括：查看课程知识图谱与节点详情；选择当前学习目标并生成个性化学习路径；查看路径总说明和单节点推荐理由；提交练习反馈并触发动态调整；撤销最近一次练习反馈；进入细化学习页面进行局部范围内的路径规划；查看推荐资源、学习者画像与近期学习记录；导出当前学习路径结果。

系统维护者的主要操作包括：初始化数据库与演示数据；恢复演示初始状态；通过冒烟脚本检查健康检查、路径生成、练习反馈提交、反馈回退与画像读取等核心链路是否正常。

三、系统功能设计

1、系统架构设计

系统整体采用“前端展示层 + 后端业务层 + SQLite 数据层”的三层结构。前端主要由首页、学习图谱、推荐资源、学习者画像和细化学习等页面组成；后端采用 `Controller -> Service -> Repository / GraphBuilder / Planner / Explainer` 的分层调用方式，分别处理接口接收、业务逻辑、数据访问、图谱构建、路径规划和推荐说明生成；数据库负责存储课程结构、学习者状态、反馈记录以及资源浏览行为。

在该架构下，前端不直接参与路径计算，只负责输入采集与结果展示；后端算法层不直接访问数据库，而是处理已经装配好的内存图结构，从而保证职责清晰；推荐理由模块必须消费规划阶段输出的依据，确保推荐解释与路径规划结果保持一致。该架构既满足当前系统实现需要，也为后续功能扩展预留了较好的维护空间。

2、系统流程设计

系统运行流程如下：学习者首先进入系统查看课程级知识图谱，了解知识点结构及依赖关系；随后在图谱页或首页选择学习目标，并结合当前掌握度和时间预算生成个性化学习路径；系统输出本轮推荐学习内容、下一轮建议和整体推荐理由；学习者完成练习后提交练习反馈，系统更新掌握度并重新规划路径；若学习者误操作或希望恢复上一状态，还可以撤销最近一次反馈；此外，学习者还可以进入细化学习页面，针对某一一级知识点继续开展二级范围内的路径规划，并查看对应推荐资源与学习画像。

这一流程形成了“目标设定 -> 路径生成 -> 练习执行 -> 练习反馈调整 -> 再规划”的学习闭环，有助于增强系统的个性化、动态性和可解释性。

3、系统功能设计

（1）知识图谱展示与目标设定  
系统支持展示数据结构课程的知识图谱，当前已具备课程级图谱、二级细化图谱和多组三级细化子图。学习者可以点击节点查看其章节、难度、说明、直接前置知识点和后继知识点，也可以在图谱页直接将节点设为当前学习目标，驱动首页重新规划学习路径。

（2）个性化学习路径规划  
系统根据学习目标、学习者掌握度和时间预算生成个性化路径，输出“本轮推荐学习”和“下一轮建议”两部分结果。路径规划遵循知识点依赖关系，优先补齐前置基础，同时兼顾掌握度差距和时间成本，使推荐结果更贴近实际学习过程。

（3）推荐理由解释  
系统不仅返回路径结果，还输出整体说明和单节点 explanation。整体说明用于回答“本次路径为什么这样安排”，单节点 explanation 用于说明某个知识点为什么被放在当前阶段。推荐理由直接消费规划阶段生成的依据，因此具有较好的可解释性。

（4）练习反馈与动态调整  
学习者完成练习后可以提交练习反馈，反馈内容包括完成状态和自评掌握度。系统会根据练习反馈更新学习者掌握度，并重新生成当前学习路径，从而反映新的学习进展。前端还支持“已完成并掌握”“看过但不熟”“未完成”等快捷反馈方式，提高操作效率。

（5）反馈回退与路径前后对比  
系统支持按最近一次反馈批次进行回退。回退后会恢复上一版掌握度，并展示回退前后路径变化，便于学习者修正误点或误判。该功能增强了系统的可恢复性和演示稳定性。

（6）细化学习导航  
对于支持继续拆分的一级知识点，系统可以进入细化学习页面，在二级范围内继续进行图谱浏览、目标选择、局部路径规划和局部练习反馈。当前系统已完成多个二级 scope 的细化支持，并覆盖多组三级细化子图，形成“课程级导航 + 细化范围导航”的双层学习路径体系。

（7）推荐资源展示  
系统在路径结果基础上，为当前节点推荐配套学习资源。资源页会展示资源类型、适合阶段、聚焦点、来源说明和选择原因；当当前节点没有直接资源时，系统还能按“节点直配资源 -> 焦点资源 -> 最近上层节点通用资源”的顺序补齐推荐，保证学习者能够获得可用资源。

（8）学习者画像展示  
学习者画像页展示当前掌握度分布和近期学习记录。学习者在首页或细化学习页提交练习反馈后，画像页中的数据会同步更新，从而形成持续积累的学习状态记录。

（9）学习路径导出与演示重置  
系统支持将当前学习路径导出为文本文件，导出内容包括目标、时间预算、路径总说明、本轮安排、下一轮建议和资源摘要，方便后续复习与论文附录整理。同时，系统支持一键恢复演示初始状态，便于重复进行答辩演示和截图采集。

四、数据库设计

1、数据库 E-R 关系说明

本系统数据库采用 SQLite，本地部署简单，便于单机开发和演示。数据库中，`courses` 表用于存储课程信息；`knowledge_points` 表用于存储课程中的知识点；`knowledge_dependencies` 表用于描述知识点之间的前置依赖关系；`learners` 表用于记录学习者信息；`learner_mastery` 表与 `learner_detail_mastery` 表分别记录课程级和细化范围下的掌握度；`learning_feedback_records` 表与 `detail_learning_feedback_records` 表分别记录课程级和细化范围下的反馈历史；`learning_resource_view_records` 表用于记录资源查看行为。

这些数据表共同支撑了知识图谱构建、学习路径规划、动态调整、资源联动与画像展示等核心业务，形成了完整的数据闭环。

（此处插入图11：数据库 E-R 图）

2、关键数据表设计

（1）courses  
该表用于保存课程基础信息，关键字段包括 `code`、`name`、`description`、`target_audience` 和 `created_at`。当前演示课程为“数据结构”。

（2）knowledge_points  
该表用于保存课程知识点，关键字段包括 `code`、`name`、`chapter_no`、`chapter_name`、`difficulty_level`、`estimated_minutes`、`display_order` 和 `description`。该表支撑知识图谱展示、路径规划和资源定位。

（3）knowledge_dependencies  
该表用于描述知识点之间的依赖关系，关键字段包括 `prerequisite_point_id`、`target_point_id`、`relation_type` 和 `description`。该表是学习路径规划的核心依据之一，保证系统生成的学习顺序符合前置关系约束。

（4）learners  
该表用于保存学习者基础信息，关键字段包括 `code`、`name`、`major`、`grade_label`、`target_course_id` 和 `note`。当前系统采用默认演示学习者进行展示。

（5）learner_mastery  
该表用于保存学习者对课程级知识点的掌握度，关键字段包括 `learner_id`、`knowledge_point_id`、`mastery_score` 和 `updated_at`。路径规划时会综合读取该表中的掌握度数据。

（6）learning_feedback_records  
该表用于保存课程级练习反馈记录，关键字段包括 `feedback_batch_id`、`completion_status`、`self_rated_mastery`、`previous_mastery`、`updated_mastery`、`rule_applied` 和 `recorded_at`。该表既支持动态调整，也支撑反馈回退与学习记录展示。

（7）learner_detail_mastery  
该表用于保存细化学习场景下的掌握度数据，关键字段包括 `scope_code`、`node_code`、`mastery_score` 和 `updated_at`。该表为二级细化学习页面中的局部路径规划提供数据基础。

（8）detail_learning_feedback_records  
该表用于保存细化学习范围内的练习反馈记录，关键字段与课程级反馈表相对应，主要用于支持细化学习中的反馈更新、历史记录和局部进度调整。

（9）learning_resource_view_records  
该表用于记录学习者资源浏览行为，关键字段包括 `resource_title`、`resource_url`、`resource_type`、`resource_source`、`recommended_phase`、`scope_code`、`linked_reason_summary` 和 `interaction_type`。该表可为后续资源推荐优化和学习行为分析提供依据。

五、系统主要功能的实现

1、课程级知识图谱与目标选择

系统已实现课程级知识图谱可视化展示，学习者可以在图谱页面中查看当前课程的知识结构，并通过点击节点查看其基础信息、前置关系和后继关系。对于支持进一步拆分的节点，系统还支持双击进入下一级知识图谱，实现逐层下钻。学习者在图谱页选中目标知识点后，可以直接将其设为当前学习目标，系统随后自动跳转到首页并生成对应路径。

2、首页个性化学习路径生成

首页已实现学习路径规划面板。系统会结合当前学习目标、学习者掌握度和可用时间预算，输出本轮推荐学习内容、下一轮建议和路径总说明。相较于静态章节推荐，该模块更突出“依赖关系约束 + 掌握度差距 + 时间预算控制”的综合规划思想，是本项目最核心的功能之一。

3、练习反馈、动态调整与回退

在学习者完成一轮练习后，可以在首页直接提交练习反馈，系统会根据练习反馈更新掌握度并重新计算学习路径，前端同步展示调整前后差异和最近一次反馈摘要。如果学习者误操作，还可以通过回退功能恢复最近一次反馈前的状态。该部分实现了学习导航从“静态推荐”到“动态更新”的闭环。

4、细化学习导航

系统已实现细化学习页面。学习者可以从首页路径项进入细化学习范围，在二级图谱中选择局部目标并生成细化路径。当前系统已支持多个二级范围的细化学习，并且对部分内容继续提供三级子图展示，使系统能够在课程级路径之外继续细化到具体知识点层面，增强导航精度。

5、推荐资源展示

系统已实现与路径联动的资源推荐页面。学习者可以从首页路径项或细化学习页面进入资源页，查看与当前节点相关的文本或视频资源，并了解每条资源的适用阶段、聚焦点和选择原因。当细化节点缺少直接资源时，系统也能通过上层继承方式提供兜底推荐，从而保证学习过程不中断。

6、学习者画像展示

学习者画像页面已实现掌握度展示、画像聚合统计与近期学习记录汇总。通过该页面，学习者可以直观了解当前课程中的学习状态变化，也便于教师观察系统是否真正记录了学习过程和结果。

7、学习路径导出与演示支持

为了增强系统的可复用性和答辩演示稳定性，系统还实现了学习路径导出、一键演示启动、一键演示检查和演示状态重置等功能。当前项目已具备固定截图清单、固定导出样例和演示脚本，便于中期检查、论文写作与答辩准备统一口径。
